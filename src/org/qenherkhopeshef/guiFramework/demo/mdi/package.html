Multiple document framework

This part of the Qenherkhopeshef framework is dedicated to multiple document applications. We test it with a simple document system, i.e. basically a text. Note that we actually use a multiple windows interface. The demo includes a number of likely features in a software:
  • a palette/tool system
  • a task/swingworker interface (the text loading process being artificially slow to demonstrate its use)
  • a system-aware interface: Mac will get a different behaviour and different menus.

Architectural notes

About menus: A menubar can apparently be linked with only one Frame. As we want to have multiple frames (especially for the mac), it means that each frame must have a menu. Considering the mac, this menu is attached to any frame, as the Mac menu is in fact the application menu, not the individual frames'. So, for instance, a dialog frame should have a menu (at least on the mac), which allows to open new documents and the like.

Hence: there needs to be one menu per JFrame. This menu must access both the document edited in the JFrame and the general application.
